# 進階功能

## 計分與速度加分

標準 1000 分、雙倍 2000 分，乘上答對比例。

**速度加分**（答對越快分數越高）有**兩層**開關：

| 位置 | 標籤 | 作用 |
| --- | --- | --- |
| 右側「這一頁」 | 這一題要不要速度加分 | 只覆寫這一題：跟隨整份／這題開／這題關 |
| 右側「整份簡報」 | 速度加分（所有題目的預設） | 整份簡報的預設值 |

單題設定優先。開啟時最快作答保留 100%、時間用完剩 50%，中間線性遞減；關閉時答對就是滿分。
不限時的題目不會套用速度加分。海龜湯與猜圖題用階段計分取代速度加分。

## 公布答案由主持人決定

計時結束**只會停止收答案**（畫面顯示「時間到」），不會自動揭曉。
答案一律等主持人按下「公布答案」才公布，也可以隨時「+15 秒」重新開放作答。

## 開放問題與文字雲

開放問題可即時彙整成文字雲，或列成回覆列表。

![即時文字雲](/img/screenshots/08-word-cloud.png)

## 自訂表情符號

管理端可以上傳自己的表情符號（最多 12 個）。參與者點一下，大螢幕就會從下方浮出來、
飄升幾秒後淡出，**不會顯示是誰送的**——匿名大家才敢按。

![匿名表情符號](/img/screenshots/09-reactions.png)

- 廣播內容只有 `id` / `url` / 流水號三個欄位，不含任何身分資訊
- 每人每 700ms 最多送一個，避免洗版
- 主持人可以隨時關閉整個功能

## 封面圖

列表、題庫市集、還有**大廳等人的時候**都用同一張。規則只有一條：

1. 作者在「整份簡報 → 封面」自己指定的那張
2. 沒指定就自動抓**第一張有配圖的題目**

自動抓的時候只看 `slide.image`，**不碰解說圖** —— 解說圖常常直接畫著答案，
拿它當封面等於在列表上爆雷。

## 按鈕音效

主持人控制列的按鈕可以有音效。音檔放在 **`data/sounds/`**（不是 `public/`）——
換一顆音效只要換檔案，不用重新 build。檔名固定，副檔名 `.mp3` / `.ogg` / `.wav` / `.m4a` / `.flac` 都認：

| 檔名 | 哪一顆按鈕 |
| --- | --- |
| `advance.*` | 主要按鈕：開始活動／下一題／下一頁 |
| `reveal.*` | 公布答案（沒放這個檔就用 `advance.*`） |
| `back.*` | 上一步 |
| `stage.*` | 揭露下一階段／給下一條提示 |
| `addtime.*` | ＋15 秒 |
| `leaderboard.*` | 看排行榜 |

沒放的檔案就是沒有聲音。資料夾第一次開機會自動建好，裡面附一份 `讀我.txt`。

## 背景音樂（大廳音樂／作答音樂）

兩首各自獨立，都在編輯器的 **整份簡報** 裡設定，都**只在主持人的大螢幕播**。

| | 什麼時候響 | 怎麼開始 |
| --- | --- | --- |
| **大廳音樂** | 等待參與者加入時 | 主持人在大廳按播放鍵；開始出題就自動收掉 |
| **作答音樂** | 開始出題後整場循環 | 自動接上；主持人可用控制列的音符鍵隨時關掉 |

作答音樂**碰到音樂題自動讓路**，離開那一頁再自己接回去；頒獎畫面會停下來把場子留給掌聲；
音量預設 35%，建議 30–40%。

## 外觀與自訂背景

介面走**扁平純色**路線，沒有任何漸層。六種底色：石板藍、石墨黑、海軍藍、梅紫、森綠、淺色。

可以上傳自己的背景圖，上傳當下前端會用 canvas 取樣分析亮度與細節，
**自動決定要壓多暗、糊多少**，確保題目文字始終清楚（亮度越高遮罩越濃、細節越多模糊越重）。
算完的值會顯示在滑桿上，想自己調也可以。

## 參與者身分：Session 識別

跟 Kahoot 一樣不需要帳號，身分綁在「這一場活動」上：

- 加入時伺服器發一組不可猜測的 **session token**，只回給本人，不會出現在任何廣播資料裡。
- 帶著 token 重連 → 拿回原本的暱稱、分數與作答紀錄（重新整理、切網路、手機休眠都沒問題）。
- **不能**用別人的 player id 冒充身分——排行榜上看得到的 id 不是憑證。

## 防作弊

- 送到參與者手機的題目**不含正解**，公布後才下發。
- 配對題右欄用每人專屬的匿名 token，不是配對組的 id。
- 猜圖題的**圖不下發**、音樂題的**音檔不下發**、數字題的**答案不下發**、大廳／作答音樂**只在主持端播**。
- **分數等公布答案才入帳**；`myAnswer` 的對錯與得分也是公布後才下發（海龜湯例外，那是它的玩法）。
- 排序題送到大螢幕的項目是**打亂過**的；分類題的項目只送 `text`，`categoryId` 就是答案。
- 上傳一律用檔案本身的 magic bytes 判斷型別，不信任副檔名或 client 給的 MIME。
- **所有批改都在伺服器端完成**，前端只負責顯示。
